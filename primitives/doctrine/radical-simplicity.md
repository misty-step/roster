# Radical Simplicity (the hundred-line koan)

Operator-ratified 2026-07-06. Code is a liability: delete as much as possible
while maintaining maximum value. We are prototyping for ourselves — never pay
code complexity for backwards compatibility; migrations delete the old path in
the same cut.

Every component collapses toward four layers: a tiny deterministic spine
(policy, persistence, gates — carrying a mechanism budget), declarations
(identities, specs, schemas), model judgment at semantic seams, and shared
renderers. Imperative lines are the residue of judgment not yet moved to a
declaration or a model. Prompts and skills are programs: prose mass is code
mass — point, don't enumerate.

In practice: the best diff is net-negative; a growing dispatch brief means a
missing declaration (fix the layer, not the brief); an organ that can't state
its one question in one sentence is two organs, or zero.
