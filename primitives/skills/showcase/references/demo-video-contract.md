# Demo Video Contract

Demo videos sell the product moment. They also rot quickly. Pin the run.

## Inputs

- Commit or release identifier.
- Demo command, fixture, seed/reset path, or replay artifact.
- Target audience and one desired action.
- Script with timestamps or scene beats.
- Capture viewport, browser/device, and any env vars.

## Formats

| Format | Use |
|---|---|
| 30-60 second silent loop | Landing hero, social preview, portfolio thumbnail |
| 90 second narrated demo | Prospect understands value without a call |
| 3-5 minute walkthrough | Technical buyer or collaborator inspection |
| Evidence replay | Internal proof, release note, QA artifact |

## Script Rules

- Show the product within the first 5 seconds.
- Show one hard edge: failure, risk, recovery, verification, or before/after.
- Narrate outcomes, not UI labels.
- Keep implementation detail for the walkthrough, not the hero loop.
- End with proof or next action, not a vague tagline.

## Verification

- Re-run the demo command from clean state.
- Capture final video plus source screenshots/transcript when possible.
- Watch once as a prospect: is the job obvious with no repo context?
- Watch once as a technical buyer: is the proof inspectable?
- Record unresolved risks: flaky setup, stale fixture, private data redaction,
  unreadable text, inaccessible motion, or unsupported public assertion.
