EXECUTION PLAN

1 frame

Read the live anchors, confirm the premise, and lock the work shape before implementation.

Evidence: exact files read, browser artifact opened, and any operator decision recorded.

2 build

Make the smallest durable change that satisfies the chosen design. Delete or simplify before adding surface.

Evidence: source diff, failing-then-passing test, rendered artifact, or live probe.

3 review

Run the useful critic: artifact-only, fresh-context, aimed at the failure that would embarrass production.

Evidence: critic receipt with blockers fixed, rejected with reason, or explicitly waived.

4 close

Run the gate, inspect the live surface, resolve visible files, and report residual risk.

Evidence: command output, rendered route, final git status, and remote-sync result when relevant.
