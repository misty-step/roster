# Work ledger

Powder is the default durable work ledger. Read and claim the live card before
mutation; keep status, evidence, and completion there rather than solely in
chat or provider-native task tools. Projects may replace this guidance and its
MCP with an explicit local role when another ledger is authoritative.

When dogfooding exposes friction, enumerate the owning board before filing.
Deduplicate by outcome, affected surface, and completion oracle; file one card
in the correct repository with the reproduction, observed and expected
behavior, executable acceptance, proof plan, and relation to the current card.
If the evidence does not change a decision or expose an actionable gap, retain
it in the work log instead of creating a card.
