# Roster v0.2.0 public acceptance

This is the redacted, durable acceptance record for Roster's first public
release. It distinguishes reproducible public automation from the one
credentialed host check that cannot run in GitHub Actions.

- Tag and commit: `v0.2.0` at
  `b9a3be0d2e83fef605bc6edadee6a3dc38e62020`.
- Successful release workflow:
  <https://github.com/misty-step/roster/actions/runs/29432786361>.

exercised installed archives before publication on Linux and macOS, published

install, init, check, show, resolve, dry dispatch, and reinstall path after
publication. Both prepublication and post-publication jobs reject development
or private-library paths.

## Clean-host and authenticated dispatch

On 2026-07-15, the published Apple Silicon archive was downloaded into a new
temporary root, verified against `checksums.txt` and GitHub's attestation, and
installed into a custom empty prefix. The test removed the ambient
`ROSTER_CONFIG` variable before invoking the installed binary, matching a
stranger's shell rather than the parent Roster session.

Measured from creation of that temporary root:

- installed binary plus new workspace config ready: 54 seconds;
- first public-only `roster resolve amos`: 104 seconds;
- authenticated `roster dispatch amos` launched: 119 seconds;
- exact sentinel `PUBLIC_ROSTER_DISPATCH_OK`, followed by a clean `/exit`:
  312 seconds total.

The longer authenticated timing includes model response time and an operator
delay: bracketed paste inserted the sentinel request into the Codex TUI, then a
separate Enter submitted it. That is Harness/operator friction, not installer
or resolver time.

## Fail-closed history and independent review

Eight non-successful release attempts preceded the ninth, successful run. They
failed closed while the release path acquired, in order: the required
changelog contract, normalized Landmark version input, portable Ubuntu build
dependencies, a bundled musl linker, canonical changelog grounding, synthesis
model fallback, tag-free initial-release history, and an absolute evidence
output path. The fifth attempt was cancelled after exposing a dependency hang;
the other seven failed. No GitHub Release existed until the ninth run passed.

A fresh read-only `scully` critic, launched through Roster from a neutral
directory with only public GitHub surfaces and the acceptance oracle, accepted
the release mechanics but rejected v0.2.0's README because its
rollback-to-absent sentence hard-coded the default prefix. It also required
this authenticated-dispatch record to be public. The critic exited cleanly;
the retained transcript and receipt hashes are:

```text
transcript bfbb84f9f0d659f9dcc98a40e6306352b8bafd8b0dcd1e96bb86ae948669ed9f
receipt    8a4283dd5c7bc8d7d013a92ab2f880ab24dcfafa58e71880740918f22fd15237
```

Published tags are not rewritten. The rollback documentation correction and
the first true previous-version rollback replay therefore ship as v0.2.1.
