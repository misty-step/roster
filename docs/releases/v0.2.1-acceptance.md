This is the redacted, durable acceptance record for the v0.2.1 corrective
release. It covers the public rollback oracle introduced after the v0.2.0
cold review and the credentialed dispatch check that cannot run in GitHub
Actions.

## Independent clean-prefix replay

On 2026-07-15, the published Apple Silicon archive was downloaded into a new
temporary root. Its checksum matched `checksums.txt`; `gh attestation verify`
bound it to the v0.2.1 tag, the release workflow, and commit `844a462`. The
archive was installed into an empty custom prefix, then its installed binary
created a new workspace config without a checkout or private source.

With ambient `ROSTER_CONFIG` explicitly removed, the installed binary reported
`roster 0.2.1`; `roster check` accepted 733 public primitive files and one
starter agent; show, resolve, and dry dispatch succeeded. A path scan of the
workspace and bundle found no operator development checkout, private
declaration root, or home-library source.

The first replay attempt deliberately does not count as evidence: it inherited
`ROSTER_CONFIG` from the parent agent session, resolved the operator's home
configuration, and was rejected when the source-path scan exposed that fact.
The corrected replay removed the variable before check, show, resolve, and
dispatch. This distinction keeps the acceptance claim tied to the exercised
public surface.
