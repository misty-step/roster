Get started

Install one binary. Declare the agents you want. Roster loads only the selected role when you launch.

Install

version=0.2.0
case "$(uname -s)-$(uname -m)" in
  Darwin-arm64) target=aarch64-apple-darwin ;;
  Darwin-x86_64) target=x86_64-apple-darwin ;;
  Linux-x86_64) target=x86_64-unknown-linux-musl ;;
  Linux-aarch64|Linux-arm64) target=aarch64-unknown-linux-musl ;;
esac
gh release download "v$version" --repo misty-step/roster \
  --pattern "roster-v${version}-${target}.tar.gz" --pattern checksums.txt
archive="roster-v${version}-${target}.tar.gz"
expected=$(grep " $archive$" checksums.txt | awk '{print $1}')
actual=$(shasum -a 256 "$archive" | awk '{print $1}')
test -n "$expected" && test "$actual" = "$expected"
tar -xzf "$archive"
"roster-v${version}-${target}/install.sh"
export PATH="$HOME/.local/bin:$PATH"

mkdir my-roster-workspace && cd my-roster-workspace
roster init
roster check

The complete install recipe selects the exact target, verifies SHA-256 and optional GitHub provenance, and includes the rollback path.

Roster does not sync a global skill catalog or rewrite permanent Harness settings.

Understand the declaration

A role owns one additive include list over guidance, skills, MCP declarations, and packs. An agent binds that role to a name, model, Harness, and safe native arguments. Any agent may dispatch any other agent through the Roster CLI.

agents:
  amos:
    description: Codex orchestrator for broad software work.
    role: core/role:orchestrator
    model: gpt-5.6-sol
    reasoning: high
    harness: codex
    args: [--search]

Launch

roster                         # pick one effective agent
roster list                    # workspace roster
roster show amos               # exact composition
roster dispatch amos --dry-run # exact projection and command
roster dispatch amos           # launch it
roster inspect amos            # config, graph, receipts

The nearest .roster/config.yaml fully replaces your home config, so a client workspace can expose a completely separate roster and MCP set without ambient globals.

Inspect the bundle

roster resolve amos --output /tmp/amos
find /tmp/amos -maxdepth 3 -type f
# AGENTS.md, selected skills, MCP declarations, manifest.yaml

Runtime bundles are ephemeral by default. Use dispatch --keep-bundle when the resolved projection itself is evidence.
