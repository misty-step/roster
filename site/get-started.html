Get started

One machine, push-button to verified-live. Every command below is copied straight from this repo's README.md and SKILL.md — nothing invented.

Install

Clone the repo and install the four binaries roster ships: the CLI, the MCP face, the HTTP API face, and the Claude hook helper. 

git clone https://github.com/misty-step/roster && cd roster
cargo install --locked --path crates/roster-cli     # `roster`
cargo install --locked --path crates/roster-mcp     # `roster-mcp` (MCP face)
cargo install --locked --path crates/roster-api     # `roster-api` (HTTP face)
cargo install --locked --path crates/roster-hooks   # `roster-hooks` (Claude hooks)
roster sync                                          # converge the managed Tier 1 core

roster sync exposes the full skill catalog and installs the orchestrator through native Tier 1 projections for Claude Code, Codex, and OMP. Other identities remain explicitly materializable. Every managed file, link, or config block is reversible with roster sync --disable.

Verify it went live

roster check                      # catalog gate: frontmatter, paths, index, markers
roster doctor --live --json       # read-only Tier 1 state + bounded live probes
roster list                       # all agents parse
claude -p "state your session identity per your loaded instructions"
# → "Session Identity: orchestrator (roster)"

Day-to-day verbs

Once installed, these are the commands you'll actually run:

roster list                                  # who exists
roster show cerberus                         # one declaration, prompt-native
roster materialize --harness codex cerberus  # per-harness install/brief form
roster brief cerberus --card <powder-id>     # dispatch brief with a live card folded in

Both binaries read the checkout via --root <path> (CLI) or the ROSTER_ROOT env var (MCP server); the default is the current directory.

MCP face

roster-mcp exposes four tools, listed in this repo's SKILL.md:

  • list — list roster agents from role.yaml declarations.
  • show — show one agent declaration as prompt-native text.
  • brief — render a dispatch brief, with optional add_skills / add_mcps overrides.
  • materialize — render one declaration for claude, codex, bb, or omp.

HTTP API face

roster-api serves the same four verbs over HTTP, bound to loopback by default: 

roster-api --root . --port 4101
curl -s http://127.0.0.1:4101/v1/agents
curl -s http://127.0.0.1:4101/v1/agents/cerberus
curl -s "http://127.0.0.1:4101/v1/agents/sweep/brief?add_skill=extra-skill"
curl -s "http://127.0.0.1:4101/v1/agents/cerberus/materialize?harness=codex"
