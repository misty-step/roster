AGENT DECLARATIONS FOR THE FACTORY

Every agent is a file, not a config buried in a harness.

Roster is the single place every agent Misty Step runs gets declared: role, model policy (preferred model plus fallbacks and a reasoning tier), permissions, skills, MCP servers, and the evidence bar it's held to — one role.yaml plus one instructions.md per agent, under agents/<name>/. The roster CLI turns a declaration into a running agent on any harness: Claude Code, Codex, Pi, or Bitterblossom's remote runners.

Every claim below is a real capture from this repo's own roster binary and its committed agent declarations — linked in the gallery, not asserted.

Why it matters

shape Data + prose, never code. proof A committed role.yaml per agent.
composition Override at dispatch. reach One declaration, every harness.
sync One lead agent syncs. fit Aesthetic tokens with the Roster mark.

Roster succeeds harness-kit by inverting the sync model: harness-kit synced the entire primitive pack to every harness on every machine, so every session carried every skill whether it needed it or not. Roster syncs one default lead agent with a curated subset; everything else — primitives/skills, primitives/mcps, the rest of the roster — is exposed through the five faces (CLI, MCP, API, SDK, skill) and discovered ad hoc, only when a dispatch actually needs it.
